# Deployment Fix Plan

## Issue
The deployment is failing because the `pnpm-lock.yaml` file is out of sync with `package.json`. Specifically:
- 9 dependencies were added to `package.json` but are missing from the lockfile
- 1 dependency (`@types/bcryptjs`) has a version mismatch between lockfile (`^3.0.0`) and manifest (`^2.4.6`)

This causes the `pnpm install` command to fail with `ERR_PNPM_OUTDATED_LOCKFILE` in the CI environment where `frozen-lockfile` is enabled by default.

## Solution
Update the lockfile to match the current `package.json` by running `pnpm install`, then commit the updated lockfile.

## Steps
1. Run `pnpm install` to update the lockfile and install dependencies
2. Verify that the lockfile now matches the package.json
3. Commit the updated `pnpm-lock.yaml` file
4. Push the changes to trigger a new deployment

## Verification
After updating the lockfile, the build should proceed past the dependency installation step.

## Notes
- This is a standard procedure when package.json is updated but lockfile is not committed
- The lockfile should always be committed to ensure reproducible builds
- No changes to the build configuration are needed
